01 Créer le dépôt GitHub Gratuit

Commencez par créer votre dépôt distant sur GitHub, puis liez-le à votre dossier local.

  • Créer le dépôt sur github.com

    Allez sur github.com/new → Nom : fullstack-cours → Public → Pas de README → Créer.

  • Initialiser Git dans votre dossier local
    # Dans le dossier qui contient tous vos fichiers
cd chemin/vers/fullstack-cours

git init
git branch -M main
BASH
  • Configurer votre identité Git (première fois uniquement)
    git config --global user.name "Votre Nom"
git config --global user.email "votre@email.com"
BASH
  • Relier votre dossier local au dépôt distant
    git remote add origin https://github.com/VOTRE-USERNAME/fullstack-cours.git

# Vérifier la connexion
git remote -v
# → origin  https://github.com/VOTRE-USERNAME/fullstack-cours.git
BASH
02 Premier push — tous les fichiers Git

Ajoutez tous vos fichiers et faites votre premier commit.

Assurez-vous que votre dossier contient bien : index.html, cours-html.html, cours-css.html, cours-tailwind.html, cours-javascript-v2.html, README.md, .gitignore et le dossier portfolio-spa/. 
# 1. Vérifier l'état — voir les fichiers non suivis
git status

# 2. Ajouter TOUS les fichiers
git add .

# 3. Vérifier ce qui sera committé
git status

# 4. Premier commit
git commit -m "init: ajout des 4 modules de cours + portfolio SPA"

# 5. Pousser vers GitHub
git push -u origin main

# ✅ Vos fichiers sont maintenant visibles sur github.com/VOTRE-USERNAME/fullstack-cours
BASH
Après ce premier push, les commandes suivantes seront simplement git add . → git commit -m "message" → git push
03 Activer GitHub Pages Gratuit · HTTPS auto

GitHub Pages publie automatiquement votre dépôt en tant que site web statique. Parfait pour les cours HTML/CSS/Tailwind/JS.

  • Aller dans les paramètres du dépôt

    Sur GitHub → votre dépôt → onglet Settings (en haut à droite).

  • Activer Pages

    Menu gauche → Pages → Section Build and deployment → Source : Deploy from a branch → Branch : main → Dossier : / (root) → Cliquer Save.

  • Attendre 1-2 minutes

    GitHub compile et déploie votre site. Rafraîchissez la page Settings → Pages.

  • Votre site est en ligne !

    URL générée automatiquement :

    https://VOTRE-USERNAME.github.io/fullstack-cours/

# Les cours sont accessibles à :
https://VOTRE-USERNAME.github.io/fullstack-cours/cours-html.html
https://VOTRE-USERNAME.github.io/fullstack-cours/cours-css.html
https://VOTRE-USERNAME.github.io/fullstack-cours/cours-tailwind.html
https://VOTRE-USERNAME.github.io/fullstack-cours/cours-javascript-v2.html
https://VOTRE-USERNAME.github.io/fullstack-cours/portfolio-spa/
URL
Mise à jour automatique : chaque fois que vous faites un git push, GitHub Pages redéploie automatiquement votre site en 1-2 minutes.
Limitation : GitHub Pages héberge du contenu statique. La partie json-server du portfolio SPA ne fonctionnera pas directement — voir l'étape 5 pour déployer le serveur.
04 Alternative — Netlify Gratuit · Drag & Drop

Netlify est une alternative très populaire à GitHub Pages. Deux méthodes au choix.

Méthode A — Drag & Drop (sans CLI)

  • Créer un compte

    Allez sur netlify.com → Sign up → Connecter avec GitHub.

  • Déployer par glisser-déposer

    Dashboard → Sites → Zone de drop en bas de page → Glissez votre dossier fullstack-cours/. Netlify génère une URL immédiatement.

Méthode B — Via CLI (avec mise à jour automatique)

# Installer la CLI Netlify
npm install -g netlify-cli

# Se connecter
netlify login

# Dans votre dossier du projet
cd fullstack-cours

# Premier déploiement
netlify deploy --prod --dir .

# URL générée :
# → https://votre-nom-aleatoire.netlify.app

# Mettre à jour après modifications
netlify deploy --prod --dir .
BASH

Méthode C — Déploiement continu depuis GitHub

  • Connecter GitHub

    Netlify Dashboard → Add new site → Import an existing project → GitHub → Sélectionner fullstack-cours.

  • Configurer

    Build command : laisser vide · Publish directory : . · Cliquer Deploy site.

  • Déploiement automatique

    Chaque git push déclenche automatiquement un redéploiement sur Netlify.

05 Déployer json-server sur Render Node.js requis

Render.com permet d'héberger gratuitement des services Node.js. C'est la solution pour avoir votre API json-server accessible en ligne.

  • Créer un compte sur render.com

    Allez sur render.com → Sign Up → Connecter avec GitHub.

  • Créer un Web Service

    Dashboard → New + → Web Service → Connecter votre dépôt fullstack-cours.

  • Configurer le service
    Name:            portfolio-api
Root Directory:  portfolio-spa
Runtime:         Node
Build Command:   npm install
Start Command:   npm start
Plan:            Free
CONFIG
  • Récupérer l'URL et mettre à jour portfolio.js
    // Dans portfolio-spa/js/portfolio.js
// Remplacer :
const API_URL = 'http://localhost:3000/projets';

// Par l'URL Render :
const API_URL = 'https://portfolio-api-xxxx.onrender.com/projets';
JS
  • Committer et pousser
    git add portfolio-spa/js/portfolio.js
git commit -m "config: URL API Render en production"
git push
BASH
Note Render gratuit : le service s'endort après 15 minutes d'inactivité. Le premier appel après une période d'inactivité peut prendre 30-60 secondes (cold start). C'est normal sur le plan gratuit.

06 Workflow Git au quotidien Bonnes pratiques

Une fois le dépôt configuré, voici les commandes Git à utiliser régulièrement.

# ── Séquence standard après chaque modification ──
git add .                           # ajouter tous les changements
git status                          # vérifier ce qui sera committé
git commit -m "type: description"   # committer
git push                            # pousser sur GitHub (déploiement auto)

# ── Commandes utiles ──
git log --oneline                   # historique des commits
git diff                            # voir les modifications en cours
git status                          # état des fichiers
git pull                            # récupérer les derniers changements

# ── Convention de nommage des commits ──
git commit -m "init: initialisation du projet"
git commit -m "feat: ajout cours HTML module 01"
git commit -m "fix: correction lien navigation"
git commit -m "style: amélioration design page accueil"
git commit -m "docs: mise à jour README"
git commit -m "refactor: refactorisation portfolio en Tailwind"
git commit -m "chore: ajout .gitignore"
BASH

Plan de commits recommandé pour ce projet

# Commit 1 — Structure
git add .gitignore README.md
git commit -m "init: structure du dépôt et documentation"

# Commit 2 — Page d'accueil
git add index.html
git commit -m "feat: page d'accueil hub des modules"

# Commit 3 — Module 01
git add cours-html.html
git commit -m "feat: cours HTML module 01"

# Commit 4 — Module 02
git add cours-css.html
git commit -m "feat: cours CSS module 02"

# Commit 5 — Module 03
git add cours-tailwind.html
git commit -m "feat: cours Tailwind CSS module 03"

# Commit 6 — Module 04
git add cours-javascript-v2.html
git commit -m "feat: cours JavaScript module 04"

# Commit 7 — Portfolio SPA
git add portfolio-spa/
git commit -m "feat(spa): application portfolio SPA complète"

# Pousser tout d'un coup
git push -u origin main
BASH
