Section 01

Présentation de HTML

HTML (HyperText Markup Language) est le langage standard utilisé pour créer et structurer le contenu d'une page Web. Il ne s'agit pas d'un langage de programmation, mais d'un langage de balisage : il décrit la structure et le sens du contenu (titres, paragraphes, liens, images…).

Définition officielle (W3C)
HTML est un langage de description de documents hypertexte conçu pour être affiché dans un navigateur Web. La version actuelle est HTML5, standardisée en 2014 et continuellement enrichie.

Historique rapide

Section 02

Concepts fondamentaux

a) Élément HTML

Un élément HTML est l'unité de base du HTML. Il représente une partie du contenu ou de la structure d'une page. Un élément est généralement composé d'une balise ouvrante, d'un contenu et d'une balise fermante. 

<!-- Anatomie d'un élément HTML -->
<p>Ceci est le contenu de l'élément.</p>
  ↑              ↑                      ↑
  Ouvrante    Contenu             Fermante
HTML

Il existe également des éléments vides (void elements) qui n'ont pas de contenu ni de balise fermante :

<br>          <!-- Saut de ligne -->
<hr>          <!-- Ligne horizontale -->
<img src="photo.jpg" alt="Photo">   <!-- Image -->
<input type="text">  <!-- Champ de saisie -->
HTML

Éléments de bloc vs éléments en ligne :

Bloc (block)En ligne (inline)
Prend toute la largeur disponibleOccupe seulement son contenu
Commence toujours sur une nouvelle ligneS'insère dans le flux de texte
<p>, <div>, <h1>, <ul>…<span>, <a>, <strong>, <img>…

b) Balise HTML

Une balise HTML (tag) est une instruction de balisage entourée de chevrons < >. Les balises indiquent au navigateur comment interpréter et afficher le contenu. 

<!-- Balise ouvrante -->
<h1>

<!-- Balise fermante (avec le slash /) -->
</h1>

<!-- Exemples de balises courantes -->
<h1>Titre principal</h1>
<h2>Titre secondaire</h2>
<p>Paragraphe de texte</p>
<strong>Texte en gras</strong>
<em>Texte en italique</em>
<a href="#">Un lien</a>

c) Attribut HTML

Les attributs HTML fournissent des informations supplémentaires sur les éléments. Ils sont toujours spécifiés dans la balise ouvrante, sous la forme nom="valeur". 

<!-- Syntaxe générale -->
<balise attribut1="valeur1" attribut2="valeur2">contenu</balise>

<!-- Exemples concrets -->
<a href="https://example.com" target="_blank" title="Visiter le site">
  Cliquez ici
</a>

<img src="logo.png" alt="Logo de la société" width="200" height="80">

<input type="email" name="email" placeholder="Votre adresse e-mail" required>
HTML

Attributs universels (utilisables sur tous les éléments) :

id="identifiant"

Identifiant unique dans la page. Utilisé par CSS et JS.

class="nom-classe"

Classe(s) CSS. Peut être partagée par plusieurs éléments.

style="propriété: valeur"

Styles CSS inline directement sur l'élément.

title="texte"

Info-bulle affichée au survol de l'élément.

lang="fr"

Langue du contenu de l'élément (accessibilité).

hidden

Cache l'élément visuellement et pour les lecteurs d'écran.

d) Structure d'une page HTML

Toute page HTML valide respecte une structure de base standardisée. Voici le squelette minimal d'un document HTML5 : 

<!DOCTYPE html>
<html lang="fr">

  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Titre de la page</title>
    <link rel="stylesheet" href="style.css">
  </head>

  <body>
    <header>
      <h1>Mon Site Web</h1>
      <nav>…</nav>
    </header>

    <main>
      <section>
        <h2>Section principale</h2>
        <p>Contenu de la page…</p>
      </section>
    </main>

    <footer>
      <p>&copy; 2025 Mon Site Web</p>
    </footer>

    <script src="script.js"></script>
  </body>

</html>
HTML
Explication des éléments clés

<!DOCTYPE html> — Déclare que le document est en HTML5. Toujours en première ligne.

<html lang="fr"> — Racine du document. L'attribut lang indique la langue pour les lecteurs d'écran et les moteurs de recherche.

<head> — Contient les métadonnées (non affichées) : encodage, titre de l'onglet, liens CSS…

<meta charset="UTF-8"> — Encodage des caractères. UTF-8 supporte tous les caractères (accents, symboles…).

<body> — Tout ce qui est visible dans le navigateur se place ici.

e) Publier du texte

HTML offre de nombreuses balises pour structurer et mettre en forme le texte d'une page.

<!-- Les 6 niveaux de titres -->
<h1>Titre de niveau 1 (un seul par page)</h1>
<h2>Titre de niveau 2</h2>
<h3>Titre de niveau 3</h3>
<h4>Titre de niveau 4</h4>
<h5>Titre de niveau 5</h5>
<h6>Titre de niveau 6</h6>

<!-- Paragraphe -->
<p>Un paragraphe est un bloc de texte. Le navigateur gère les
retours à la ligne automatiquement selon la largeur disponible.</p>

<!-- Mise en forme sémantique -->
<p>Voici du texte <strong>en gras (important)</strong>,
du texte <em>en italique (emphase)</em>,
du texte <mark>surligné</mark>,
et du texte <del>barré</del>.</p>

<!-- Retour à la ligne et séparateur -->
<p>Première ligne.<br>Deuxième ligne.</p>
<hr> <!-- Ligne séparatrice horizontale -->

<!-- Citation -->
<blockquote>
  Le secret de la réussite, c'est de ne jamais décrocher.
</blockquote>

<!-- Code informatique -->
<p>Utilisez la balise <code>&lt;code&gt;</code> pour du code inline.</p>
HTML
Attention : N'utilisez pas les balises de mise en forme pour leurs effets visuels, mais pour leur sens sémantique. Par exemple, <strong> signifie "important" (pas juste "gras"). Le style visuel est du ressort du CSS.

  </li>

  </li>

g) Lier des contenus

Les liens hypertextes sont au cœur du Web. La balise <a> (ancre) permet de créer des liens vers d'autres pages, fichiers, sections, ou adresses e-mail. 

<!-- Lien vers une page externe -->
<a href="https://www.google.com" target="_blank">
  Aller sur Google
</a>

<!-- Lien vers une page interne du même site -->
<a href="contact.html">Page de contact</a>

<!-- Lien vers une section de la même page (ancre) -->
<a href="#section-about">À propos</a>
<!-- ... plus bas dans la page -->
<section id="section-about">...</section>

<!-- Lien vers une adresse e-mail -->
<a href="mailto:contact@example.com">Envoyer un mail</a>

<!-- Lien vers un numéro de téléphone -->
<a href="tel:+221771234567">+221 77 123 45 67</a>

<!-- Lien de téléchargement -->
<a href="cv.pdf" download>Télécharger mon CV</a>

<!-- Image cliquable (lien contenant une image) -->
<a href="https://example.com">
  <img src="logo.png" alt="Logo">
</a>
HTML
Valeur de targetComportement
_selfOuvre dans l'onglet actuel (défaut)
_blankOuvre dans un nouvel onglet
_parentOuvre dans le cadre parent
_topOuvre dans la fenêtre principale

h) Afficher une image

La balise <img> est un élément vide (pas de balise fermante) qui permet d'intégrer une image dans la page. Elle nécessite deux attributs essentiels : src et alt. 

<!-- Syntaxe de base -->
<img src="images/photo.jpg" alt="Description de l'image">

<!-- Avec dimensions et titre -->
<img
  src="images/projet.png"
  alt="Capture d'écran du projet portfolio"
  width="800"
  height="450"
  title="Projet Portfolio — 2025"
>

<!-- Image depuis une URL externe -->
<img src="https://picsum.photos/400/300" alt="Image aléatoire">

<!-- Image responsive (CSS) -->
<img
  src="hero.jpg"
  alt="Image héro"
  style="max-width: 100%; height: auto;"
>

<!-- Figure avec légende (bonne pratique HTML5) -->
<figure>
  <img src="projet.jpg" alt="Mon projet">
  <figcaption>Fig.1 — Aperçu du projet réalisé en 2025</figcaption>
</figure>
HTML
FormatExtensionUsage recommandé
JPEG.jpg / .jpegPhotos, images complexes avec dégradés
PNG.pngImages avec transparence, logos, screenshots
GIF.gifAnimations simples
SVG.svgIllustrations vectorielles, icônes
WebP.webpFormat moderne : qualité optimale + petite taille
L'attribut alt est obligatoire. Il décrit l'image pour les lecteurs d'écran (accessibilité) et s'affiche si l'image ne peut pas être chargée. Pour les images purement décoratives, utilisez alt="" (chaîne vide).

i) Présenter un formulaire

Les formulaires HTML permettent de collecter des données saisies par l'utilisateur. La balise <form> est le conteneur principal, et les différents champs de saisie sont créés avec <input>, <textarea>, <select>, etc. 

<form action="traitement.php" method="POST">

  <!-- Champ texte simple -->
  <label for="nom">Nom du projet :</label>
  <input type="text" id="nom" name="nom"
         placeholder="Ex : Portfolio Web" required>

  <!-- Zone de texte multilignes -->
  <label for="description">Description :</label>
  <textarea id="description" name="description"
             rows="5" cols="50"
             placeholder="Décrivez votre projet..."></textarea>

  <!-- Téléchargement de fichier (image) -->
  <label for="image">Image du projet :</label>
  <input type="file" id="image" name="image"
         accept="image/*">

  <!-- Menu déroulant -->
  <label for="categorie">Catégorie :</label>
  <select id="categorie" name="categorie">
    <option value="">-- Choisir --</option>
    <option value="web">Web</option>
    <option value="mobile">Mobile</option>
    <option value="desktop">Desktop</option>
  </select>

  <!-- Cases à cocher -->
  <fieldset>
    <legend>Technologies utilisées :</legend>
    <label><input type="checkbox" name="tech" value="html"> HTML</label>
    <label><input type="checkbox" name="tech" value="css"> CSS</label>
    <label><input type="checkbox" name="tech" value="js"> JavaScript</label>
  </fieldset>

  <!-- Boutons -->
  <button type="submit">Ajouter le projet</button>
  <button type="reset">Réinitialiser</button>

</form>

Modélisation d'un Projet
Un projet est caractérisé par : libellé, description, technologies utilisées (liste), image, et optionnellement une date de réalisation et un lien de démo.

🚀 Mise en place GitHub

Avant de commencer à coder, créez votre dépôt distant et initialisez Git localement :

# 1. Créer le dépôt sur GitHub : "fullStack_portfolio" (public)
#    → https://github.com/new

# 2. Initialiser le dépôt local
git init fullStack_portfolio
cd fullStack_portfolio

# 3. Configurer le dépôt distant
git remote add origin https://github.com/VOTRE-USERNAME/fullStack_portfolio.git

# 4. Créer la structure de fichiers
mkdir images
touch lister-projets.html detailler-projet.html ajouter-projet.html

# 5. Premier commit
git add .
git commit -m "init: structure de base du projet portfolio"
git push -u origin main
