Section 01

Présentation de Tailwind CSS

Tailwind CSS est un framework CSS utility-first (utilitaire en premier) : plutôt que de fournir des composants pré-stylisés (comme Bootstrap), il propose des centaines de petites classes CSS atomiques que l'on compose directement dans le HTML pour construire n'importe quelle interface.

Utility-First — Le principe fondamental
Chaque classe Tailwind correspond à une et une seule déclaration CSS. On assemble ces briques directement dans l'attribut class de chaque élément HTML, sans jamais écrire de CSS personnalisé (dans la majorité des cas).

— CSS Traditionnel vs Tailwind

CSS Traditionnel
/* style.css */
.carte {
  background-color: #1a1a24;
  padding:          24px;
  border-radius:    12px;
  border:           1px solid #2e2e40;
  box-shadow:       0 4px 20px 0
                    rgba(0,0,0,.4);
}
.carte:hover {
  transform: translateY(-4px);
}
/* index.html */
<div class="carte">...</div>
Tailwind CSS
<!-- Pas de CSS séparé ! -->

<div class="
  bg-slate-800
  p-6
  rounded-xl
  border border-slate-700
  shadow-xl
  hover:-translate-y-1
  transition-transform
">
  ...
</div>

— Avantages & Inconvénients

Avantages ✅Inconvénients ⚠️
Pas besoin de nommer des classes CSS (fin du problème de nommage) Attribut class HTML plus verbeux et long
CSS généré = seulement les classes utilisées (très léger en production) Courbe d'apprentissage initiale (mémoriser les utilitaires)
Pas de conflit de styles, tout est colocalisé dans le HTML Peut rendre le HTML difficile à lire pour les novices
Personnalisation facile via tailwind.config.js Nécessite un outil de build (Node.js) dans le workflow standard
Design system intégré (palette de couleurs, échelle d'espacement cohérente) Réutilisation de styles → recourir à @apply ou aux composants
Responsive & dark mode intégrés via préfixes Moins adapté aux projets avec beaucoup de CSS unique complexe

Section 02

Mise en place de l'environnement

Il existe plusieurs façons d'intégrer Tailwind CSS dans un projet. Nous couvrons les deux méthodes principales : la CDN Play (idéale pour apprendre) et l'installation via npm (recommandée pour la production).

01 Méthode CDN — Play (sans installation)

La façon la plus rapide de démarrer avec Tailwind. Parfait pour le prototypage et les cours. Il suffit d'ajouter un seul <script> dans le <head> de la page.

La méthode CDN est réservée au développement et au prototypage. Elle ne génère pas de fichier CSS optimisé et ne supporte pas la personnalisation avancée. En production, utilisez toujours l'installation npm. 
<!DOCTYPE html>
<html lang="fr">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Mon Portfolio</title>

  <!-- ✅ Tailwind CSS via CDN Play (développement uniquement) -->
  <script src="https://cdn.tailwindcss.com"></script>

</head>
<body class="bg-gray-900 text-white">

  <h1 class="text-4xl font-bold text-amber-400">
    Mon Portfolio
  </h1>

</body>
</html>
HTML

02 Méthode npm — Installation locale (production)

L'installation via npm permet de générer un fichier CSS optimisé contenant uniquement les classes utilisées dans votre projet (purge automatique). 

# Pré-requis : Node.js installé (https://nodejs.org)

# 1. Initialiser un projet npm dans le dossier du portfolio
npm init -y

# 2. Installer Tailwind CSS et ses dépendances
npm install -D tailwindcss postcss autoprefixer

# 3. Générer le fichier de configuration Tailwind
npx tailwindcss init

# → Crée tailwind.config.js à la racine du projet
BASH
// tailwind.config.js — Configuration de base
/** @type {import('tailwindcss').Config} */
module.exports = {
  // 👇 Indiquer TOUS les fichiers HTML/JS qui utilisent des classes Tailwind
  content: [
    "./*.html",           // toutes les pages HTML à la racine
    "./src/**/*.{html,js}" // fichiers dans /src
  ],
  theme: {
    extend: {
      // Personnalisations ici (Section 3j)
    },
  },
  plugins: [],
}
JS
/* src/input.css — Fichier CSS source (3 directives obligatoires) */
@tailwind base;        /* Reset CSS de Tailwind (Preflight) */
@tailwind components;   /* Classes composants (@apply) */
@tailwind utilities;    /* Toutes les classes utilitaires */
CSS
# 4. Lancer le compilateur en mode surveillance (watch)
npx tailwindcss -i ./src/input.css -o ./dist/output.css --watch

# Tailwind surveille les fichiers HTML, recompile dès qu'une classe change
# → Génère dist/output.css avec SEULEMENT les classes utilisées
BASH
<!-- Lier le CSS compilé dans toutes les pages HTML -->
<head>
  <link rel="stylesheet" href="dist/output.css">
</head>
HTML
Structure du projet avec Tailwind :
fullStack_portfolio/
├── tailwind.config.js — configuration
├── package.json — dépendances npm
├── src/input.css — fichier CSS source (directives @tailwind)
├── dist/output.css — CSS compilé (lié dans les HTML)
├── lister-projets.html
├── detailler-projet.html
└── ajouter-projet.html

03 Méthode CDN avec configuration (Tailwind v3+)

Pour utiliser la CDN tout en personnalisant la configuration (couleurs, polices…), il est possible d'injecter la configuration directement dans la page via un script : 

<script src="https://cdn.tailwindcss.com"></script>
<script>
  tailwind.config = {
    theme: {
      extend: {
        colors: {
          'portfolio-accent': '#7c3aed',
          'portfolio-fond':   '#0f0f13',
        },
        fontFamily: {
          'titre': ['Georgia', 'serif'],
        }
      }
    }
  }
</script>

Section 03

Concepts fondamentaux

Convention de nommage Tailwind : la plupart des classes suivent le patron [propriété]-[valeur] ou [propriété]-[couleur]-[nuance]. Exemples : text-xl, bg-blue-500, p-4, border-2. Les préfixes comme hover:, md: ajoutent des conditions.

a Couleurs

Tailwind intègre une palette de couleurs complète et harmonieuse organisée en familles de 11 nuances (50 → 950). Chaque couleur peut être appliquée au texte, à l'arrière-plan, aux bordures, aux ombres, etc.

Syntaxe générale :
[propriété]-[famille]-[nuance]
Exemples : text-amber-400, bg-slate-900, border-rose-500
Slateslate-50 → slate-900
Blueblue-50 → blue-900
Greengreen-50 → green-900
Amberamber-50 → amber-900
Roserose-50 → rose-900
Violetviolet-50 → violet-900

g Modificateur

Les modificateurs (ou variants) sont des préfixes qui conditionnent l'application d'une classe. On les applique avec la syntaxe [modificateur]:[classe].

hover:

Au survol de la souris

focus:

Quand l'élément est sélectionné (clavier/clic)

active:

Pendant le clic/appui

disabled:

Quand l'élément est désactivé

group-hover:

Quand le parent .group est survolé

peer-focus:

Quand le frère .peer est en focus

dark:

En mode sombre (prefers-color-scheme)

first: / last:

Premier / dernier enfant

odd: / even:

Enfants impairs / pairs

placeholder:

Texte placeholder d'un input

before: / after:

Pseudo-éléments ::before / ::after

checked:

Checkbox/radio cochée

j Personnalisation

La puissance de Tailwind réside dans sa personnalisabilité totale via tailwind.config.js. On peut étendre (ou remplacer) n'importe quelle valeur du design system par défaut. 

// tailwind.config.js — Personnalisation complète du portfolio
module.exports = {
  content: ["./*.html"],
  darkMode: 'class',          // mode sombre via class="dark"
  theme: {
    extend: {               // EXTEND = ajoute SANS écraser le défaut

      /* Couleurs personnalisées */
      colors: {
        'portfolio': {
          'fond':    '#0f0f13',
          'surface': '#1a1a24',
          'accent':  '#7c3aed',
          'accent2': '#06b6d4',
        }
      },

      /* Polices */
      fontFamily: {
        'titre': ['Georgia', 'Times New Roman', 'serif'],
        'code':  ['Courier New', 'monospace'],
      },

      /* Espacements supplémentaires */
      spacing: {
        '18': '4.5rem',   // 72px — non dispo dans Tailwind de base
        '88': '22rem',
      },

      /* Border radius personnalisé */
      borderRadius: {
        'card': '12px',
      },

      /* Animations */
      animation: {
        'fade-in': 'fadeIn 0.5s ease-in-out',
        'slide-up': 'slideUp 0.3s ease-out',
      },
      keyframes: {
        fadeIn: {
          '0%':   { opacity: '0' },
          '100%': { opacity: '1' },
        },
        slideUp: {
          '0%':   { transform: 'translateY(20px)', opacity: '0' },
          '100%': { transform: 'translateY(0)',    opacity: '1' },
        },
      },

      /* Largeur max du conteneur */
      maxWidth: {
        'portfolio': '1100px',
      },
    },
  },
  plugins: [],
}
